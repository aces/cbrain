Files


Viewing Files

The Files tab, called the file manager, is a link to an interface that displays a list of files, file details and a file management menu bar at the top of the page. The file management menu bar consists of seven tabs. More details can be displayed for each file by selecting the Toggle Details button. Jiv Files are shown as links under files with Jiv files as children resulting from a minc2jiv operation. Additionally Files highlighted in pink are unavailable. Each individual file has a details page that can be accessed by clicking on that particular file.

The synchronization symbols, which list the status of a file, are shown at the bottom of the files page and include:
  • ProvNewer
    • Indicates that no known files on cache side or files on data provider side are known to be newer (default when there are no SyncStatus object at all).
  • CacheNewer
    • Indicates that the file on cache side is known to be newer than on data provider.
  • InSync
    • Indicates that the file contains a synched version of the data providerâ€™s content.
  • ToCache
    • Indicates that the data provider's file is being copied to cache.
  • ToProvider
    • Indicates that the cached file is being copied to data provider.
  • Corrupted
    • Indicates that a transfer to a provider was never completed.

Sidebar

The sidebar provides the options to Toggle Details, Show all files on the system and Tree View. The Toggle Details button allows the user to see more details of the files. Tree View gives the user the ability view the parent and child files hierarchically.
A user can search by filename. Additionally, Active Filters lists the filters currently applied to the files which can be removed by clicking on the 'x' beside the applied filter.

Filter Files Drop Down

The filter files drop down allows a user to filter their files by type or by tag (shown below). A tag is a marker that can be created and be used to mark or "tag" a file. Files can be searched by filters and tags. This drop down allows a user to filter files either by the preset filters or by custom filters.



A user can also create a custom filter as shown below. This allows a user to filter by filename, creation date, size, owner, group, data provider and tags.




Upload File
The Upload File drop down in the file management menu, which allows a user to upload individual files and compressed files with sizes up to 250MB. The interface is shown below. You can browse for these files on your local computer, assign tags to the uploaded file, groups and group file permissions, archive options (for zip and tar files) and CBRAIN will store the file. Note that the compressed files cannot have folders within it. Compressed files have the option to be saved as it is, extracted or a file collection of the compressed files can be generated. The preferred method for uploading files to CBRAIN is through the data providers tab.





Help
The help drop down provides help for files, tools, information on file formats and Jiv viewer information.

File Menu Bar


Downloading Files
To download files, select the files and click download files. If there is more than one file then a tarball containing the files is created.

Organizing Files
The tags feature shown below under the Update Attributes button provides the ability to organize files. It allows a user to apply customized tags to files in order to organize them and make them easily searchable (see below). Just select the file, the tag and click on the button Update Tags.



The Update Attributes button also provides the ability to modify files tags and projects. Projects can be changed by selecting the file and clicking Update Projects. Project permissions can also be updated by selecting the file along with selecting the desired project permission.

Groups
Groups facilitate the sharing of files between users. There are four types of groups which are all subsets of projects and include: system groups, site groups, user groups and work groups.
  • System Groups
    • Every user on CBRAIN is automatically a member of the system group everyone.
  • Site Groups
    • Each site has a site group automatically associated with it.
  • User Groups
    • Every user on CBRAIN is automatically assigned a group, that only they are a member of, specified by that user's username.
  • Work Groups
    • A work group is a type of group and also a project that can be created by anyone.
File permissions are managed through projects. Projects allow sharing of files between members of a project. Users can be members of any project that either the site manager or the CBRAIN administrator assigned to them or that they created themselves. A file can only belong to one project but a user can belong to many projects. Project permissions can be set by selecting the file, Update Files Attributes, the project and the permissions.

Creating a Collection
The Create Collection button on the files menu allows a user to select files and create a collection.

Moving and Copying Files
Files can be selected and moved or copied from one data provider to another using the Move/Copy Files button on the files menu.

Jiv Viewer
Jiv Viewer is another tool available in the file management menu under Jiv Viewer. This drop down menu provides a list of the Jiv files in the files list, which can be selected and viewed using the Jiv Viewer in a Java Applet window.

Using Tools
The Conversion Tool and Scientific Tool drop downs on the file management menu provide a drop down selection of conversion and scientific operations that can be applied to files as tasks. When a tool is selected from the drop down menu, another select box appears to specify an execution server to complete the task. Execution servers that have that particular tool available and that you have sufficient privileges to access, will be presented as possible selections along with an option to select a cluster to execute the task randomly. For some tools, you must select files and the desired tool to be brought to an option selection page. An example tool option page is shown below.


Once a task has completed, the resulting files are placed in the files list on the Files tab page. For example, once a Civet task has been run using the Civet Tool a Civet Collection is returned. Clicking on that particular resulting Civet Collection from a Civet task brings you to the file details page (see below). Through this page you can see the details of the civet collection, change the group, group permissions and update the tags associated with a file. The Contents link displays the Civet output image file. Also the link Display full CIVET file list displays all the files in the Civet Collection, which can be extracted individually.
